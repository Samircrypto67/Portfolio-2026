1. Contexte professionnel

La Maison des Ligues de Lorraine (M2L) gère plusieurs infrastructures sportives et souhaite moderniser son système de gestion des membres et des réservations de salles. L’objectif est de remplacer les tableaux Excel par une application centralisée, sécurisée et accessible par le personnel administratif.

2. Description de la situation professionnelle

Développement d’une application Java permettant :

  • Gestion des membres (ajout, modification, suppression)
  • Gestion des salles et disponibilités
  • Gestion des réservations avec vérification des conflits
  • Système d’authentification sécurisé pour les administrateurs

3. Objectifs de la mission

  • Créer un logiciel fiable et sécurisé
  • Mettre en place une base de données centralisée (Supabase PostgreSQL)
  • Développer une interface Java avec architecture MVC
  • Permettre une gestion rapide et centralisée des données
  • Tester toutes les fonctionnalités pour garantir le bon fonctionnement

4. Moyens matériels et logiciels

  • Ordinateur Windows avec IDE (IntelliJ/Eclipse)
  • Supabase PostgreSQL pour la base de données
  • Java JDK et bibliothèques JDBC
  • Git pour versionnement

5. Réalisations techniques

  • Création du schéma entité-association pour la base de données
  • Création des tables : membres, salles, réservations
  • Développement de l’application Java (MVC)
  • Connexion JDBC à Supabase PostgreSQL
  • Implémentation d’un système d’authentification sécurisé

6. Compétences mobilisées

  • Programmation Java et architecture MVC
  • Gestion et manipulation de base de données relationnelle
  • Gestion des droits d’accès et sécurité des données
  • Travail en environnement cloud

7. Résultats obtenus

L’application permet désormais une gestion centralisée et sécurisée des membres, salles et réservations, facilitant le travail des administrateurs et réduisant les erreurs humaines.

Schéma entité-association

Chaque réservation est liée à un membre et à une salle via des clés étrangères pour garantir l’intégrité des données.

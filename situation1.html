Contexte

Dans le cadre d'un besoin interne de la M2L, j'ai développé une application de gestion du personnel en Java permettant d'administrer les données des employés de manière centralisée. Cette application répond aux exigences de suivi des informations RH et de maintenance des données personnelles.

Objectif

Concevoir et développer une application Java complète permettant la gestion complète du personnel, incluant l'ajout, la modification, la suppression et la consultation des données des employés, avec une connexion sécurisée à une base de données PostgreSQL et le respect des principes de la programmation orientée objet.

Fonctionnalités principales

  • Gestion complète des employés (CRUD)
  • Système d'authentification et d'autorisation
  • Interface en ligne de commande intuitive
  • Connexion persistante à la base de données
  • Gestion des erreurs et validation des données
  • Architecture modulaire et maintenable

Environnement technique

  • Langage : Java 8
  • IDE : Eclipse IDE
  • Build tool : Maven
  • Base de données : PostgreSQL
  • Connexion : JDBC (Driver PostgreSQL)
  • Dépendances : MySQL Connector, CommandLineMenus
  • Système : Windows 11

Architecture et conception

L'application suit une architecture en couches avec séparation claire des responsabilités :

  • Couche présentation : Interface utilisateur en ligne de commande
  • Couche métier : Logique de gestion des employés
  • Couche données : Accès à la base de données via DAO pattern

Compétences mobilisées

  • Développement d'une application Java complète avec Maven
  • Connexion et manipulation d'une base de données PostgreSQL via JDBC
  • Structuration du code en programmation orientée objet (POO)
  • Implémentation du pattern DAO pour l'accès aux données
  • Gestion des exceptions et validation des entrées utilisateur
  • Tests et validation fonctionnelle de l'application
  • Documentation et organisation du code source

Résultats et livrables

L'application a été développée avec succès et permet une gestion efficace du personnel. Le code source est organisé, documenté et prêt pour une éventuelle évolution. Les tests fonctionnels confirment le bon fonctionnement de toutes les fonctionnalités implémentées.
